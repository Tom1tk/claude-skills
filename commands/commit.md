Write a commit message for the staged changes:
1. Run `git diff --staged` to read what is actually staged
2. Summarise the intent, not just the mechanics
3. Follow Conventional Commits format:
   - type(scope): short summary (≤72 chars)
   - Types: feat, fix, refactor, docs, test, chore, perf, ci
4. Add a body if the why isn't obvious from the subject line
5. Do not include "generated by AI" or similar noise

Then run `git commit -m "..."` with the message.
If nothing is staged, say so and stop.
